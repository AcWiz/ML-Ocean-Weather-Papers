# 记忆增强配置方案

> 第三阶段 - 认知架构升级

---

## 当前状态

### 已有能力
- memory-core: 基础记忆搜索工具
- memory-lancedb: LanceDB 向量存储 (需配置)
- session-memory: 会话级短期记忆

### 需要增强
- 多层级记忆管理
- 自动重要性判断
- 混合搜索能力

---

## 改进方案

### 1. 记忆层级设计

```
┌─────────────────────────────────────┐
│         长期记忆 (持久化)            │
│   - .learnings/ 学习日志            │
│   - memory/ 每日记忆                │
│   - 向量数据库 (可选)              │
└─────────────────────────────────────┘
                ↑
┌─────────────────────────────────────┐
│         中期记忆 (可回溯)           │
│   - SQLite 会话存储                 │
│   - 7天滚动窗口                    │
└─────────────────────────────────────┘
                ↑
┌─────────────────────────────────────┐
│         短期记忆 (即时)             │
│   - 当前会话上下文                  │
│   - 50轮对话缓冲                   │
└─────────────────────────────────────┘
```

### 2. 记忆检索策略

| 场景 | 检索范围 | 优先级 |
|------|----------|--------|
| 日常对话 | 短期 → 中期 | 即时 |
| 专业问题 | 短期 → 长期 → 向量 | 深度 |
| 错误处理 | .learnings/ERRORS | 紧急 |

### 3. 自动记忆触发

- **每轮对话结束**: 评估是否需要记忆
- **错误发生**: 自动记录到 ERRORS.md
- **用户纠正**: 自动记录到 LEARNINGS.md
- **重要发现**: 提示是否记录

---

## 配置建议

建议添加 memory-lancedb 插件以启用向量搜索:

```json
{
  "plugins": {
    "slots": {
      "memory": "memory-lancedb"
    }
  }
}
```
